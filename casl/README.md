## CASL

ability level이라는 개념을 사용(일종의 허가수군 같은 것이라 생각하면 될듣)
- 4 종류의 parameter 값들을 받아 정의 된다.
    - User Action : 허가/제한 할 대상이 되는 행위/행동들을 지칭 ex) delete / read 같은... (이벤트 함수 같은 것들도 넘길 수 있는 지는...)
    - Subject : Action이 적용되어지는 대상 / 대상의 타입
    - Fields : Subject의 특정 fields를 지정 할 수 있게 해주는 듯 - 
    - Conditions : Subject에 대해 실행되어지는 객체/함수를 의미하는 거 같다..

공식 문서는, role 컨셉과는 전혀 다른 것이라고는 한다..
